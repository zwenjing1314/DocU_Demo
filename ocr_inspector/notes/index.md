# :root 是 CSS 中的一个伪类选择器
## 具体作用：

1. 匹配文档的根元素
- 在 HTML 文档中，:root 选择器指向 <html> 元素。
- 它和直接使用 html { ... } 的效果几乎一样，但有一些细微差别。
2. 提高优先级（特异性）
- :root 的 CSS 特异性（specificity）比普通的 html 选择器更高。
- 这意味着在 :root 中定义的样式会优先于在 html 选择器中定义的相同属性。
常用于定义全局 CSS 变量
- 正如选中的代码所示，:root 最适合用来定义CSS 自定义属性（也叫 CSS 变量），例如：
```css
     :root {
       --bg: #efe7db;
       --panel: rgba(255, 252, 246, 0.94);
       --accent: #9a4e2e;
     }
```
这些变量可以在整个项目的其他 CSS 规则中通过 var(--bg)、var(--accent) 等方式引用。
把它们放在 :root 中可以确保这些变量在整个文档中都可用（全局作用域）。

## 为什么不用 html 而用 :root？
- 语义更清晰：看到 :root 就知道这里定义的是全局级别的配置（如主题色、字体等）。
- 优先级更高：避免被其他地方的 html 选择器意外覆盖。
- 跨文档兼容：在非 HTML 文档（如 SVG 或 XML）中，:root 仍然有效，而 html 只针对 HTML。


## 总结
在这段代码中，:root 的作用是定义一组全局可用的 CSS 颜色变量，供整个网页的其他部分统一调用，方便主题管理和样式维护。

#  * 是 CSS 中的通配符选择器（Universal Selector）
## 具体作用：
1. 匹配所有元素
- * 会选中页面中的每一个 HTML 元素（如 <div>, <p>, <span>, <body> 等）。
- 在这段代码中，它的作用是给页面上的所有元素统一设置一个基础属性。

2. 统一盒模型计算方式 (box-sizing: border-box)
- 这是现代网页开发中最常见的“重置”样式之一。
- 默认情况 (content-box)：当你给一个元素设置 width: 100px; padding: 10px; border: 5px solid black; 时，它在页面上实际占用的宽度是 100 + 10*2 + 5*2 = 130px。这会导致布局计算非常麻烦。
- 使用 border-box 后：你设置的 width: 100px 就是它最终在页面上占用的总宽度。浏览器会自动从这 100px 里减去 padding 和 border 的空间，剩下的才是内容区域。

## 总结
这段代码的意思是：“页面上所有元素的宽度和高度都包含其内边距和边框。” 这是一种让 CSS 布局更容易控制的标准化做法。

# id 和 for 属性通过建立关联来增强表单的可用性和可访问性
1. 它们各自的作用：
id="fileInput" (在 <input> 标签上)
- 唯一标识符：id 必须是当前页面中唯一的。它给这个输入框起了一个“身份证号”，方便 CSS 样式定位、JavaScript 脚本操作，或者像这里一样，被其他元素引用。
for="fileInput" (在 <label> 标签上)
- 指向目标：for 属性的值必须与某个表单元素的 id 完全一致。它告诉浏览器：“这个标签是专门用来描述那个 id 对应的元素的。”

2. 它们之间的关系与好处：
当 <label> 的 for 属性与 <input> 的 id 匹配时，会产生以下效果：
点击联动（提升用户体验）
- 用户不仅可以点击那个小小的“选择文件”按钮来触发上传，直接点击“文件”这两个字（Label 文本）也能触发同样的动作。这扩大了用户的点击区域，让操作更方便。
辅助技术支持（无障碍访问/A11y）
- 对于使用屏幕阅读器（Screen Reader）的视障用户，当他们聚焦到这个输入框时，阅读器会自动读出 <label> 里的文字（即“文件”），让他们知道这个输入框是用来做什么的。如果没有这种关联，视障用户可能无法理解该输入框的用途。

总结
它们是一对**“搭档”**：
id 是元素的名字。
for 是标签发出的呼叫。
关系：for 的值等于 id 的值，从而将文字说明和输入控件绑定在一起。

# 链式调用
## slice(0, MAX_WORD_CHIPS)：截取片段
作用：从 words 数组中复制一部分出来，而不改变原数组。
参数：0 是起始索引，MAX_WORD_CHIPS（即 60）是结束索引。
目的：防止页面上一次性渲染成千上万个单词导致浏览器卡死。这里只取前 60 个词作为“预览”。
结果：得到一个最多包含 60 个元素的新数组。

## map((word) => ...)：批量转换（核心逻辑）
作用：map 会遍历数组中的每一个元素，并对它们执行相同的操作，最后返回一个由这些操作结果组成的新数组。
里面的语法：模板字符串 (Template Literals)
你会看到代码被反引号
```javascript
 `${...}` 插值表达式：在反引号内部，任何放在 `${}` 里的内容都会被当作 JavaScript 代码执行，并把结果填入字符串中。
    例子：
        ```javascript
        class="word-chip ${chipClass}" 
        // 如果 chipClass 是 "low"，结果就是 class="word-chip low"
        
        ${escapeHtml(word.text)}
        // 把单词的实际文本填进去，并经过转义处理防止乱码或攻击
```
在模板字符串里还有一个小语法：
*   `word.line_num ?? '-'`：这是空值合并运算符。如果 `line_num` 是 `null` 或 `undefined`，就显示 `-`，否则显示原来的行号。
*   `formatConfidence(...)`：直接调用之前定义的工具函数，把数字变成百分比字符串。

## .join('')：拼接成串
*   **作用**：把数组里所有的字符串元素连接成一个长长的单一字符串。
*   **参数**：`''` 表示连接时中间不加任何分隔符。
*   **为什么要用它？**
    *   `map` 之后得到的是 `['<span>...</span>', '<span>...</span>']` 这样的数组。
    *   HTML 引擎需要的是一个完整的字符串 `'<span>...</span><span>...</span>'` 才能正确渲染。
*   **结果**：最终生成一段可以直接塞进 `innerHTML` 的完整 HTML 代码。

### 总结数据流向图：
1.  **原始数据**：`[{text: '你好', ...}, {text: '世界', ...}, ...]` (可能有几千个)
    ↓ `slice`
2.  **截取后**：`[{text: '你好', ...}, ...]` (只剩前60个)
    ↓ `map` (配合模板字符串)
3.  **转换后**：`['<span>你好</span>', '<span>世界</span>', ...]` (变成了字符串数组)
    ↓ `join`
4.  **最终产物**：`"<span>你好</span><span>世界</span>..."` (一个巨大的 HTML 字符串)

这种写法在现代前端开发中非常普遍，因为它比传统的 `for` 循环拼接字符串更简洁、更易读，也更不容易出错。
